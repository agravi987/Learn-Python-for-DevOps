# 06-Infrastructure-as-Code-Support 🏗️

Infrastructure as Code, usually called IaC, means managing infrastructure with files and automation instead of manual clicks. Python often supports IaC workflows by generating config, validating files, building dynamic inventories, and preparing templates.

- How Python supports IaC tools like Ansible and Terraform.
- How to generate dynamic inventories.

- How YAML parsing helps validate infrastructure files.
- How to build a mini inventory generator.

## 🧠 Where Python Fits in IaC

Python is not always the main IaC language, but it is excellent glue.

Common Python IaC tasks:

- Generate Ansible inventory from cloud APIs.
- Validate YAML before a deployment.
- Render config files from templates.
- Convert Terraform outputs into another format.
- Create small helpers for CI/CD infrastructure checks.
- Build custom Ansible modules.

## 🤖 Ansible + Python

Ansible itself is written in Python, and Python is often used around Ansible.

You can use Python to:

- Create dynamic inventory.
- Build custom modules.
- Generate variable files.
- Validate playbook input.
- Parse Ansible facts.

Simple inventory structure:

```yaml
all:
  children:
    web:
      hosts:
        web-01:
          ansible_host: 10.0.1.10
        web-02:
          ansible_host: 10.0.1.11
```

## 📋 Dynamic Inventory

Static inventory is written manually. Dynamic inventory is generated by code.

Why dynamic inventory matters:

- Cloud servers change frequently.
- Auto Scaling groups create and remove instances.
- Different environments need different host lists.
- Manual inventory becomes outdated quickly.

Example Python data:

```python
servers = [
    {"name": "web-01", "ip": "10.0.1.10", "group": "web"},
    {"name": "db-01", "ip": "10.0.2.10", "group": "db"}
]
```

You can turn that into Ansible YAML automatically.

## 🧩 Jinja2 Templates

Jinja2 creates text files from templates and variables. It is useful for Nginx configs, systemd units, Kubernetes manifests, and app config files.

Install:

```bash
pip install jinja2
```

Basic example:

```python
from jinja2 import Template

template = Template("server {{ name }} listens on {{ port }}")
output = template.render(name="api", port=8080)

print(output)
```

Template with a loop:

```python
from jinja2 import Template

template = Template("""
upstream backend {
{% for server in servers %}
    server {{ server }};
{% endfor %}
}
""")

print(template.render(servers=["10.0.1.10:8080", "10.0.1.11:8080"]))
```

## 🧾 YAML Parsing

Use PyYAML to load, validate, and write YAML.

```python
import yaml
from pathlib import Path

inventory = yaml.safe_load(Path("inventory.yaml").read_text())
print(inventory["all"])
```

Validation example:

```python
if "all" not in inventory:
    raise ValueError("Inventory must contain an 'all' group")
```

## 🌱 Terraform Support

Terraform can output infrastructure data as JSON:

```bash
terraform output -json
```

Python can read that JSON and pass values to another tool.

```python
import json
from pathlib import Path

outputs = json.loads(Path("terraform-output.json").read_text())
load_balancer_dns = outputs["load_balancer_dns"]["value"]
print(load_balancer_dns)
```

This is useful when CI/CD needs Terraform-created values.

## 🛠️ Mini Project: Generate an Ansible Inventory

Goal: Create a Python script that reads a list of servers and generates `inventory.yaml`.

### Step 1: Create `servers.json`

```json
[
  {"name": "web-01", "ip": "10.0.1.10", "group": "web"},
  {"name": "web-02", "ip": "10.0.1.11", "group": "web"},
  {"name": "db-01", "ip": "10.0.2.10", "group": "db"}
]
```

### Step 2: Read Server Data

```python
import json
from pathlib import Path

servers = json.loads(Path("servers.json").read_text())
```

### Step 3: Build the Inventory Dictionary

```python
inventory = {
    "all": {
        "children": {}
    }
}

for server in servers:
    group = server["group"]
    inventory["all"]["children"].setdefault(group, {"hosts": {}})
    inventory["all"]["children"][group]["hosts"][server["name"]] = {
        "ansible_host": server["ip"]
    }
```

### Step 4: Write YAML

```python
import yaml

Path("inventory.yaml").write_text(
    yaml.safe_dump(inventory, sort_keys=False)
)

print("✅ Created inventory.yaml")
```

### Step 5: Add Validation

```python
for server in servers:
    for field in ["name", "ip", "group"]:
        if field not in server:
            raise ValueError(f"❌ Missing {field} in server entry")
```

### Step 6: Improve It

- Add `ansible_user` for each group.
- Generate inventory from AWS EC2 tags.
- Add environment support: `dev`, `stage`, `prod`.
- Create a Jinja2 template for an Nginx upstream file.
- Add a CLI option for the input filename.
